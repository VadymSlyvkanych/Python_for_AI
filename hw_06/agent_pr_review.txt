Домашнее задание 6
Идея простого агента: Агент для ревью pull request'ов

1. Идея

Агент автоматически анализирует pull request в GitHub и выдаёт
код-ревью: находит потенциальные баги, проблемы безопасности,
нарушения стиля кода, утечки ресурсов и даёт рекомендации по
улучшению. Разработчику не нужно ждать человека-ревьювера для
базовой проверки.

2. Инструменты

- GitHub API (через PyGithub или httpx):
  получение diff, метаданных PR, списка изменённых файлов
- LLM (Gemini через OpenRouter):
  анализ кода, поиск ошибок, генерация замечаний
- langchain-core:
  организация цепочки вызовов: загрузка diff -> анализ ->
  формирование отчёта
- os / pathlib:
  кеширование результатов, работа с файлами

3. Как работает

Шаг 1. Разработчик передаёт агенту URL pull request'а.
       Пример: "https://github.com/user/repo/pull/42"

Шаг 2. Агент через GitHub API получает diff изменений
       (GET /repos/{owner}/{repo}/pulls/{number}/files).

Шаг 3. Агент разбивает diff по файлам. Для каждого файла
       определяет язык программирования по расширению.

Шаг 4. Каждый файл отправляется LLM с промптом:
       "Проверь код на ошибки, проблемы безопасности,
       утечки памяти, несоответствие стилю. Верни список
       замечаний с указанием строки, типа проблемы и
       рекомендации."

Шаг 5. Агент собирает все замечания в единый отчёт,
       группирует по файлам и серьёзности (critical,
       warning, suggestion).

Шаг 6. Агент выводит отчёт в консоль или публикует как
       комментарий к PR (опционально через GitHub API).

4. Пример работы

Вход:
  https://github.com/example/myapp/pull/10

Выход (отчёт):
  === Ревью PR #10: myapp ===
  Файл: src/db.py
    [CRITICAL] Строка 42: SQL-инъекция. Используй
              параметризованные запросы вместо f-string.
    [WARNING] Строка 15: Не обработан возможный KeyError
              при доступе к config["api_key"].

  Файл: src/api.py
    [SUGGESTION] Строка 78: Длинная функция (50 строк).
                 Рекомендуется разбить на несколько
                 меньших функций.

5. Потенциальные улучшения

- Добавить инструменты для запуска тестов (pytest) для
  проверки, что изменения не ломают существующий код.
- Возможность автоматически поставить approve/label на PR
  если критические ошибки не найдены.
- Интеграция с линтерами (ruff, mypy) через локальный запуск.
- Кеширование уже проверенных коммитов.
