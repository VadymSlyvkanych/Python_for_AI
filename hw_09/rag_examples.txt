RAG: примеры задач и реальные системы

Часть 1. Примеры задач, где RAG особенно полезен

1. Медицинский ассистент для парамедиков

   Проблема: Парамедики в полевых условиях должны быстро находить
   правильные дозировки лекарств, протоколы действий и местные
   правила — под давлением, без доступа к интернету или времени
   листать бумажные справочники.

   Как RAG решает:
   - Все медицинские протоколы, таблицы дозировок и местные
     инструкции загружены в векторную базу
   - Парамедик спрашивает голосом или текстом: "доза эпинефрина
     для ребёнка 20 кг"
   - RAG находит точный фрагмент протокола и выдаёт ответ
     с цитатой из источника
   - Результат: ответ за секунды, всегда по актуальной версии
     документа, с возможностью проверить источник

   Реальный пример: American Legion Ambulance (ALA) построила
   такую систему на AWS Bedrock для своих машин скорой помощи.

2. Анализ контрактов иdue diligence

   Проблема: Юристы и аналитики тратят часы на поиск нужных
   пунктов в многолетних контрактах,financial reports, PDF-файлах.
   LLM не знает содержимого конкретных документов компании.

   Как RAG решает:
   - Все контракты, отчёты и документы индексируются в векторном
     хранилище с сохранением прав доступа
   - Запрос: "найди все контракты с штрафными санкциями больше
     $50K за последние 3 года и оцени риски"
   - RAG ищет релевантные фрагменты, LLM синтезирует ответ
   - Результат: часы поиска сокращаются до минут, все ответы
     подкреплены ссылками на конкретные страницы документов

   Реальный пример: Bayer построила PRINCE — RAG-систему для
   поиска по десятилетиям исследовательских отчётов. Box AI
   делает то же для корпоративных документов 100 000+ компаний.

3. Чат-бот поддержки с актуальной документацией продукта

   Проблема: Клиенты задают вопросы про новую версию продукта,
   но LLM обучена на старых данных. Ответы могут быть
   неактуальными или опасными (например, устаревшие команды).

   Как RAG решает:
   - Документация каждой версии продукта автоматически
     индексируется в векторную БД при релизе
   - Вопрос клиента сначала ищет по документации актуальной
     версии, затем LLM формулирует ответ
   - При обновлении документации достаточно переиндексировать
     источник — переобучать модель не нужно
   - Результат: ответы всегда по текущей версии, поддержка
     работает без задержек на обновление знаний модели

   Реальный пример: DoorDash использует RAG для поддержки
   курьеров (Dasher support). Thomson Reuters — для клиентской
   поддержки. LinkedIn комбинирует RAG с графом знаний для
   ответов клиентам.
